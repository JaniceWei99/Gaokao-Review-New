# Product Manager Skill

Product management intelligence for building successful products. Includes product strategy, user research, feature prioritization, roadmap planning, and go-to-market strategies.

## Actions

- **plan**: Create product plans, strategies, and roadmaps
- **research**: Conduct user research, market analysis, and competitive analysis
- **prioritize**: Prioritize features using frameworks like RICE, MoSCoW, Kano
- **define**: Define product requirements, user stories, and acceptance criteria
- **analyze**: Analyze metrics, user feedback, and product performance
- **validate**: Validate assumptions through MVPs, prototypes, and user testing
- **launch**: Plan and execute product launches and go-to-market strategies
- **iterate**: Plan iterations, improvements, and product evolution
- **communicate**: Create product communications, stakeholder updates, and documentation
- **optimize**: Optimize conversion, retention, and key product metrics

## Product Frameworks

### Prioritization Frameworks
- **RICE**: Reach, Impact, Confidence, Effort
- **MoSCoW**: Must have, Should have, Could have, Won't have
- **Kano Model**: Basic, performance, delight features
- **Value vs Effort Matrix**: High value/low effort prioritization
- **ICE Score**: Impact, Confidence, Ease

### Product Development
- **Agile/Scrum**: Sprint planning, standups, retrospectives
- **Lean Startup**: MVP, build-measure-learn cycle
- **Design Thinking**: Empathize, define, ideate, prototype, test
- **Jobs to be Done**: User motivation and desired outcomes
- **OKRs**: Objectives and Key Results for goal setting

### User Research
- **User Interviews**: Qualitative user insights
- **Surveys**: Quantitative user feedback
- **Usability Testing**: Evaluate user experience
- **A/B Testing**: Compare feature variations
- **Persona Development**: User archetype creation

## Best Practices

1. **User-Centric**: Always start with user needs and pain points
2. **Data-Driven**: Use data to inform decisions and validate assumptions
3. **Iterative**: Build, measure, learn, and iterate quickly
4. **Cross-Functional**: Collaborate with engineering, design, marketing, and sales
5. **Stakeholder Management**: Keep stakeholders informed and aligned
6. **Prioritization**: Focus on high-impact, low-effort items first
7. **Validation**: Validate assumptions before building
8. **Communication**: Clear, transparent communication with all stakeholders
9. **Metrics**: Define success metrics upfront and track them consistently
10. **Flexibility**: Be willing to pivot based on learnings and market changes

## Common Questions

- "What should we build next?" → Use prioritization frameworks
- "How do we validate this idea?" → Create MVP or run user research
- "What's our product strategy?" → Define vision, positioning, and roadmap
- "How do we improve retention?" → Analyze churn, improve onboarding, add value
- "What features should we prioritize?" → Use RICE, MoSCoW, or value vs effort
- "How do we measure success?" → Define KPIs and track metrics
- "What do our users want?" → Conduct user research and analyze feedback
- "How do we launch this feature?" → Create launch plan and coordinate teams
- "What's our competitive advantage?" → Conduct competitive analysis
- "How do we improve conversion?" → Analyze funnel, run A/B tests, optimize UX
