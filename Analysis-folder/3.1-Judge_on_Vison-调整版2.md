

# 📊 调整版2 Product Vision 深度审查报告

---

## 一、总体评价

这一版比前两版有了**质的飞跃**——合规边界清晰、定位更聚焦、付费分层合理、风险意识增强。但仍存在**一个致命硬伤和几个需要改进的地方**。

## 二、🔴 致命硬伤：科目体系描述错误

文档中写的是：

> **科目体系**: 上海高考6+1科目（语数英+物化生政史地）

**这是事实性错误，会导致整个知识点树的设计逻辑出问题。**

### 上海高考真实科目体系

根据最新政策，2025年上海高考采用 **"3+3"模式**：

| 类别 | 科目 | 分值 | 说明 |
|------|------|------|------|
| **统一高考（大三门）** | 语文、数学、外语 | 每门150分 | 全体考生必考 |
| **等级性考试（小三门）** | 从思想政治、历史、地理、物理、化学、生命科学6门中**自选3门** | 每门70分 | 考生自主选择 |
| **总分** | — | **660分** | — |

### 关键考试时间节点

| 考试 | 时间 | 说明 |
|------|------|------|
| **1月合格考** | 1月4-5日、11-12日 | 语文、数学等科目合格性考试 |
| **1月外语考（一年两考第一次）** | 与春季考试同期 | 外语科目实行**一年两考** |
| **春季高考** | 1月 | 春季考试招生 |
| **5月等级考** | 5月5-6日 | 选考科目等级性考试（小三门） |
| **6月合格考** | 6月28-30日 | 补考/其他科目合格考 |
| **6月秋季高考** | 6月7-9日 | 统一高考（大三门） |

### ⚠️ 这个错误的影响范围

```
"6+1"错误表述
    ↓
知识点树设计为9个科目全部展示
    ↓
每个学生实际只考6门（3必考+3选考）
    ↓
家长看到9个科目的知识点树会困惑
    ↓
标记了不考的科目 = 无效劳动 = 用户流失
```

### ✅ 必须做的修改

```markdown
## 修改前：
科目体系: 上海高考6+1科目（语数英+物化生政史地）

## 修改后：
科目体系: 上海高考"3+3"模式
- 必考科目（大三门）：语文、数学、外语（每门150分）
- 选考科目（小三门）：从物理、化学、生命科学、
  政治、历史、地理中自选3门（每门70分）
- 总分：660分

## 产品设计调整：
- 新用户注册时，必须先选择孩子的3门选考科目
- 知识点树只展示"语数英 + 已选3门"共6个科目
- 支持后续修改选科组合
```

### 知识点数量表也要同步修改

```markdown
## 修改前：展示全部9科

## 修改后：按选科组合动态展示
| 类别 | 科目 | 一级 | 二级 | 三级 | 预估总数 |
|------|------|------|------|------|----------|
| 必考 | 语文 | 4-5 | 15-20 | 60-80 | 80-100 |
| 必考 | 数学 | 5-6 | 20-25 | 80-100 | 100-130 |
| 必考 | 外语 | 4-5 | 15-20 | 50-70 | 70-90 |
| 选考 | （根据学生选科动态展示3门） | ... | ... | ... | ... |

注：每位学生实际使用6个科目的知识点树
```

## 三、🔴 里程碑时间节点不完整

文档中的里程碑列表缺少了几个**关键节点**：

### 当前版本缺失的节点

| 缺失节点 | 重要程度 | 说明 |
|----------|----------|------|
| **1月外语一考** | ⭐⭐⭐⭐⭐ | 上海外语实行**一年两考**，1月第一次考试是重大决策点 |
| **5月等级考具体日期** | ⭐⭐⭐⭐⭐ | 5月5-6日，这是小三门考试日，比秋季高考还早一个月 |
| **综合评价批次** | ⭐⭐⭐⭐ | 复旦、上交大等通过综评招生，时间节点与普通高考不同 |
| **1月/6月合格考** | ⭐⭐⭐⭐ | 合格考分两个时段，文档未区分 |
| **三校生高考** | ⭐⭐⭐ | 5月举行，部分用户可能涉及 |

### ✅ 建议修改为

```markdown
### 上海高考里程碑（按时间排序）：

#### 高三上学期（9月-1月）
- 9月：开学，进入一轮复习
- 11月：期中考试
- 12月：各区一模考试
- **1月初：合格考（语数外等科目）**
- **1月中：外语一年两考-第一次考试 + 春季高考**
- **1月下：春考志愿填报**

#### 高三下学期（2月-6月）
- 3月：各区二模考试
- 4月：各区三模考试 / 综评报名
- **5月5-6日：等级考（小三门）⭐ 关键节点**
- 5月下：三校生高考
- **6月7-9日：秋季高考（大三门）⭐ 最终决战**
- 6月下：成绩公布 + 志愿填报
- 6月28-30日：合格考（补考）
- 7月：录取
```

## 四、🟡 其他需要改进的地方

### 1. 免费版"限3个科目"的限制逻辑有问题

文档写的是：

> 免费版：基础标记功能（限3个科目）

但上海高考是"3+3"模式，每个学生考6门科目。如果免费版只能标记3个科目：

- 家长可能只标记语数英（必考），选考科目无法标记
- 但**5月等级考比6月高考更早**，选考科目才是最先需要冲刺的
- 免费版可能让家长在最关键的时候"卡住"，产生负面体验

> 💡 **建议**：将付费门槛从"科目数量限制"改为"功能深度限制"：

```
免费版：全部6科基础标记 + 基础倒计时
       （验证核心价值，积累用户）

付费版：高级分析报告 + AI顾问 + 趋势预测 + 家长社区
       （数据积累后的增值服务）
```

### 2. 竞品分析章节仍然缺失

这一版仍然没有独立的竞品分析。2025年AI高考工具赛道已经非常拥挤：

| 竞品 | 规模 | 核心功能 | 与本产品的关系 |
|------|------|----------|---------------|
| **QQ浏览器高考频道** | 单季服务超5000万用户 | AI志愿填报、高考信息 | 间接竞品（志愿端） |
| **夸克高考** | 累计服务1.2亿+ | 深度搜索、志愿报告、智能选志愿 | 间接竞品（志愿端） |
| **教育部"阳光志愿"** | 官方免费平台 | 免费志愿填报服务 | 间接竞品（免费替代） |
| **腾讯元宝** | 已有"上海高考"专属智能体 | 按地区细分的AI高考助手 | 潜在直接竞品 |
| **科大讯飞AI学习机** | 硬件+软件生态 | 本地化学情分析、个性化学习 | 潜在竞品（学情分析） |

> 💡 **建议**：新增竞品分析章节，重点论证：
> - 以上竞品**集中在志愿填报**环节（高考后），本产品聚焦**备考过程管理**（高考前）
> - 以上竞品**面向学生**，本产品**面向家长**
> - 这两个差异构成了真正的**蓝海空间**

### 3. "复习建议"功能踩线风险

在"薄弱点分析视图"中有一条：

> **复习建议**: 针对薄弱点的复习建议

同时在"合规边界"部分又说不能做学科知识讲解。这两者之间的边界**非常模糊**：

```
"建议重点复习三角函数倍角公式" → ✅ 这是管理建议
"倍角公式是 sin2α = 2sinαcosα，建议..." → ❌ 这是学科讲解
```

> 💡 **建议**：将"复习建议"改为更安全的表述：

```markdown
## 修改前：
- 复习建议：针对薄弱点的复习建议

## 修改后：
- 优先级排序：基于薄弱程度×考试时间×分值权重，
  自动排序"最值得优先攻克的知识点"
- 不提供任何学科内容，只告诉家长"应该先关注什么"
```

### 4. Phase 定义存在矛盾

| 问题 | 具体矛盾 |
|------|----------|
| Phase 3 标注为"付费功能" | 但 Phase 1 的付费分析中就已设计了免费/付费分层 |
| Phase 1 包含"基础分析视图" | 但分析视图本身是 Phase 3 的"高级分析"的基础版，边界不清 |

> 💡 **建议**：重新命名各Phase的目标：

```
Phase 1：核心验证（免费，验证家长是否真的会用）
Phase 2：功能完善（仍免费，完善使用体验）
Phase 3：变现启动（推出付费版，验证付费意愿）
Phase 4：增值扩展（AI功能，提高ARPU）
```

### 5. 成功指标缺少产品质量指标

当前的KPI全部是"用户侧"指标（DAU、留存、NPS），但缺少**产品本身的质量指标**：

> 💡 **建议补充**：

| 指标 | 说明 | 目标 |
|------|------|------|
| **知识点覆盖率** | 知识点树覆盖高考考纲的比例 | >95% |
| **知识点标记准确反馈率** | 家长标记后反馈"找不到知识点"的比例 | <5% |
| **单次标记耗时** | 家长完成一次标记的平均时间 | <60秒 |
| **里程碑时间准确率** | 预设时间节点与实际时间的一致性 | 100% |

### 6. User Journey 缺少"首次使用"场景

四个场景（日常沟通、试卷分析、家长会、考前）都是日常使用场景，但**缺少最关键的"第一次使用"场景**——家长第一次打开小程序时会看到什么？怎么引导？

> 💡 **建议补充**：

```markdown
#### 0. 首次使用场景（Onboarding）
**家长在家长群看到推荐，扫码进入**

Step 1：选择孩子的年级（高一/高二/高三）
Step 2：选择孩子的3门选考科目 ⭐
Step 3：系统自动生成6科知识点树 + 对应的里程碑时间线
Step 4：引导标记第一个薄弱知识点（30秒内完成）
Step 5：展示效果预览——"坚持记录，1个月后你会看到这样的分析报告"

目标：3分钟内完成首次使用，体验到核心价值
```

## 五、✅ 这一版做得好且应保留的部分

| 亮点 | 评价 |
|------|------|
| **合规边界章节** | 非常重要的新增，AI顾问的"可以/不可以"边界清晰 |
| **付费心理分析** | "焦虑驱动、效果导向、时间价值、安心感"四个动机抓得准 |
| **付费阻力分析** | 主动识别阻力（效果不确定、使用门槛、隐私顾虑）是成熟的产品思维 |
| **知识点维护机制** | 版本管理+用户反馈+质量保证流程，考虑到了长期运营 |
| **付费转化节点** | "标记第10个薄弱点时提示升级"这类触发式转化设计非常巧妙 |
| **扩展路径** | 上海→全国的三步走战略清晰 |

> **这一版的产品思维已经非常成熟——合规意识、付费设计、风险识别都到位了。但"3+3选科模式"这个事实性错误必须立即修正，它会影响知识点树、注册流程、免费版限制逻辑等多个核心设计决策。修正这一点后，这份Vision就可以进入开发阶段了。**
